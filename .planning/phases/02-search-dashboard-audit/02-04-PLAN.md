**As a** medicine manager, **I want to** recover accidentally deleted medicines from a Trash Bin and review every change ever made to a medicine's record, **so that** I never lose data and can always audit what happened.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → Trash permanent delete | Destructive action; data cannot be recovered after permanentDeleteMedicine |
| IndexedDB → ChangeHistory display | History entries are insert-only; no user-editable history API |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-07 | Tampering | permanentDeleteMedicine — must not cascade-delete history | high | mitigate | permanentDeleteMedicine in historyOps.ts never calls db.history.delete(); acceptance criterion verifies via grep; TDD tests in 02-01 cover this |
| T-02-08 | Tampering | history entries (insert-only) | low | accept | History table has no user-facing edit endpoint; only historyOps functions write to it; all writes are in transactions |
| T-02-09 | Elevation of Privilege | Permanent delete without confirmation | medium | mitigate | AlertDialog confirmation required per UI-SPEC before permanentDeleteMedicine is called; AlertDialog blocks interaction until confirmed |
| T-02-10 | Spoofing | ChangeHistory renders history medicineName field | low | accept | medicineName is stored from DB (user entered it originally); React escapes all rendered strings; no XSS surface |
</threat_model>
