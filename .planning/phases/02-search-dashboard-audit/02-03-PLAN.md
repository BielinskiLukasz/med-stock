## Phase Goal

**As a** medicine manager, **I want to** see at a glance how many medicines are expired, expiring soon, or have exceeded their open period, **so that** I can take action immediately without scanning the full list.

<objective>
Build the Dashboard vertical slice: DashboardCard reusable component and the Dashboard route with 4 tappable alert cards. Metric computation runs in a single useLiveQuery pass over active medicines using calculateStatus(). Tapping alert cards pre-sets the Zustand filter and navigates to the Medicines screen. Replaces the stub dashboard created in Plan 02-02.

Purpose: Dashboard is the daily driver for expiry awareness. It surfaces the most urgent safety information (expired/expiring medicines) without requiring the user to scroll through the full list.

Output: src/components/DashboardCard.tsx, src/routes/dashboard/index.tsx (full implementation replacing stub).
</objective>

Replace src/routes/dashboard/index.tsx with full implementation:

Imports: `useLiveQuery` from 'dexie-react-hooks', `useNavigate` from 'react-router-dom', `db` from '@/lib/db', `calculateStatus` from '@/lib/expiry', `DashboardCard` from '@/components/DashboardCard', `useUIStore` from '@/stores/uiStore'.

Single useLiveQuery call (deps []): fetches all active medicines, computes all 4 metrics in one pass:
  `const stats = useLiveQuery(async () => { const all = await db.medicines.toCollection().filter(m => m.deletedAt === null).toArray(); const now = new Date(); const today = now.toISOString().slice(0, 10); const in30 = new Date(now.getTime() + 30 * 86_400_000).toISOString().slice(0, 10); let expired = 0, expiringSoon = 0, exceededOpenPeriod = 0; for (const m of all) { const status = calculateStatus(m, now); if (status === 'Expired') expired++; if (m.manualStatus === null && m.expiryDate && m.expiryDate > today && m.expiryDate <= in30) expiringSoon++; if (status === 'Expired' && m.expiryDate && m.expiryDate > today) exceededOpenPeriod++; } return { total: all.length, expired, expiringSoon, exceededOpenPeriod }; }, [])`.

Note on Pitfall 3: calculateStatus IS inside useLiveQuery here, which is intentional for the dashboard — the dashboard only updates when DB changes, not on timer ticks. This is the Pattern 9 approximation documented in RESEARCH.md. The trade-off is documented in code comments.

Loading guard: `if (!stats) return <div className="p-4">Loading...</div>`.

Navigate: `const navigate = useNavigate()`. Get `clearAllFilters, toggleStatus` from useUIStore.

Tap handlers:
  - Expired card: `() => { clearAllFilters(); toggleStatus('Expired'); void navigate('/medicines'); }`
  - Expiring Soon card: `() => { clearAllFilters(); void navigate('/medicines'); }` (advisory — no calculateStatus for this state)
  - Exceeded Open Period card: `() => { clearAllFilters(); toggleStatus('Expired'); void navigate('/medicines'); }`

JSX layout:
  Page header: `<h1 className="text-xl font-semibold p-4">Dashboard</h1>`
  2x2 grid: `<div className="grid grid-cols-2 gap-4 p-4">`. Four DashboardCard children:

    2. Expired: label="Expired", count=stats.expired, colorClass='bg-red-50 border-red-200 text-red-700', onTap=expired tap handler
    3. Expiring Soon: label="Expiring Soon (30 days)", count=stats.expiringSoon, colorClass='bg-amber-50 border-amber-200 text-amber-700', onTap=expiring soon handler
    4. Exceeded: label="Exceeded Open Period", count=stats.exceededOpenPeriod, colorClass='bg-orange-50 border-orange-200 text-orange-700', onTap=exceeded handler

Export as: `export function DashboardScreen()`.
