## Phase Goal

**As a** household medicine manager, **I want to** have every medicine change automatically recorded and soft-deletion routed through a Trash Bin, **so that** I never lose data accidentally and can always audit what changed.

<objective>
Establish the data foundation for all Phase 2 features: Dexie schema v2 migration (deletedAt field + history table) and the historyOps.ts utility module covering all medicine lifecycle mutations with atomic transactions.

Purpose: Every subsequent Phase 2 plan (Search, Dashboard, Trash, History) depends on the schema migration being correct. The historyOps.ts functions are the single point of truth for all medicine mutations that must record history. Getting this right first prevents cascading bugs across all other plans.

Output: db.ts with version(2) block, historyOps.ts with all lifecycle functions, historyOps.test.ts with TDD coverage of all functions.
</objective>
