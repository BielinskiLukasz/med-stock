## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → Dexie write | Medicine name, expiry date, notes fields cross from unvalidated DOM input into IndexedDB |
| Service worker → browser cache | Precached asset manifests are generated at build time and signed by Workbox; no runtime tampering path |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | Medicine form inputs (name, notes) | medium | mitigate | React auto-escapes all JSX interpolation; never use dangerouslySetInnerHTML in any component — enforced via ESLint rule react/no-danger |
| T-01-02 | Tampering | Dexie write operations (db.medicines.add) | medium | mitigate | Wrap all Dexie writes in try/catch; catch block logs to console and shows user-friendly message — raw Dexie errors never propagated to UI (ASVS V7) |
| T-01-03 | Spoofing | Service worker serving stale JavaScript | low | mitigate | registerType: 'autoUpdate' in VitePWA causes new SW to activate after navigation; Workbox injects content hash into precache manifest so stale assets are never served |
| T-01-04 | Denial of Service | IndexedDB storage quota exhaustion | low | accept | `navigator.storage.persist()` increases the quota limit; no quota guard needed for Phase 1 (1,000 medicine records is well within 50 MB minimum quota) |
| T-01-SC | Tampering | npm install (vite, dexie, react-router-dom, etc.) | high | mitigate | All packages pre-audited in RESEARCH.md §Package Legitimacy Audit; all SUS verdicts confirmed as false positives (too-new heuristic); no SLOP packages detected |
</threat_model>
