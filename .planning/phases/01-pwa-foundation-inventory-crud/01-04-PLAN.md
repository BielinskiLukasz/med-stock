<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → db.locations.add() | Location name text crosses from unvalidated DOM input into IndexedDB |
| deleteLocation → db.medicines.modify() | locationId from UI state is used to identify and update medicines — cascade scope must be bounded to the correct location name |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | Location name input (add, rename) | medium | mitigate | Trim whitespace before storage; min-length validation (reject empty string) in addCustomLocation and renameLocation; React JSX auto-escapes the name in all display contexts |
| T-04-02 | Tampering | deleteLocation cascade scope | medium | mitigate | Dexie transaction('rw', db.locations, db.medicines) ensures the cascade is atomic — if any step fails, the entire transaction rolls back; no partial state possible |
| T-04-03 | Tampering | isDefault bypass (client-side only guard) | low | accept | Guard exists in locationOps.ts server logic; UI also hides Edit/Delete for isDefault locations; no backend to enforce server-side, but single-device local app means no external attacker |
| T-04-04 | Information Disclosure | Raw Dexie errors in UI | low | mitigate | All locationOps calls wrapped in try/catch in component; only user-friendly message shown; raw error logged to console only |
</threat_model>
