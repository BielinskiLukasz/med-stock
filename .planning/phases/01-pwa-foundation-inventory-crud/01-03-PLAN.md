<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Form submit → db.medicines.add() / .update() | All user-entered text (name, notes, category) crosses from DOM input into IndexedDB |
| URL params → Dexie query | :id from URL is parsed as Number() and passed to db.medicines.get() — invalid id returns undefined |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | Medicine form text fields (name, notes, category) | medium | mitigate | React JSX interpolation auto-escapes all string values; dangerouslySetInnerHTML is never used in any component in this plan |
| T-03-02 | Tampering | :id URL param coercion (Number(id)) | low | mitigate | useLiveQuery returns undefined for non-existent id; component renders "Not found" state rather than erroring; no SQL injection possible with Dexie primary key get |
| T-03-03 | Tampering | Dexie update without ownership check | low | accept | Single-device local-only app; no multi-user; any device user can edit any record — this is the intended behavior for a household app |
| T-03-04 | Information Disclosure | Raw Dexie errors in UI | low | mitigate | All db.medicines.add() and db.medicines.update() calls wrapped in try/catch; catch block shows generic user-friendly message; raw error logged to console only |
</threat_model>
